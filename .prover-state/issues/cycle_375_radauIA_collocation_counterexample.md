# Issue: Radau IA collocation-family bridge is false as stated

## Blocker

The cycle-375 target theorem
`IsCollocation ∧ HasRadauIANodes → IsAlgStable` is false under the live project
definition of `ButcherTableau.IsCollocation`.

The live `IsCollocation` predicate requires `C(s)`. The genuine collocation
tableau on the left-endpoint Radau nodes `{0, 2/3}` satisfies `IsCollocation`
and has nodes that are zeros of `algStabilityBoundaryPoly 2 (-1)`, but it is
not algebraically stable.

## Context

A concrete counterexample is the 2-stage left-Radau collocation tableau

```lean
noncomputable def leftRadau2Collocation : ButcherTableau 2 where
  A := ![![0, 0], ![1 / 3, 1 / 3]]
  b := ![1 / 4, 3 / 4]
  c := ![0, 2 / 3]
```

It satisfies:

```lean
leftRadau2Collocation.IsCollocation
leftRadau2Collocation.HasRadauIANodes
¬ leftRadau2Collocation.IsAlgStable
```

The algebraic-stability failure is immediate from the coordinate vector
`v = e₀`: the quadratic form reduces to `M₀₀ = -1/16`.

This was checked in a temporary scratch file during cycle 375; the essential
code and conclusions are recorded above.

## What was tried

- Read `OpenMath/Adjoint.lean`, `OpenMath/Symmetry.lean`, and
  `OpenMath/ReflectedMethods.lean`.
- Verified that `ReflectedMethods.reflect` transfers `B`, `C`, `D`, and `E`,
  but its algebraic-stability matrix is the negative of the original matrix,
  so it cannot provide an algebraic-stability bridge.
- Checked the live Radau IA concrete tableaux. They are not `C(s)`
  collocation tableaux: `rkRadauIA2` has `C(1)`/`D(2)`, and `rkRadauIA3`
  has `C(2)`/`D(3)`.
- Submitted five focused Aristotle jobs after the sorry-first fallback
  skeleton. After the required 30-minute wait, the node jobs were still in
  progress and the counterexample jobs completed with errors, so no returned
  proof was incorporated.

## Possible solutions

- Replace the target family theorem with a Radau IA theorem using the
  Radau IA simplifying-assumption shape, e.g. `B(2*s - 1)`, `C(s - 1)`,
  and `D(s)`, rather than `IsCollocation`.
- Alternatively, introduce a separate theorem for the concrete Radau IA family
  generated by the Radau IA tableau construction, not arbitrary collocation
  methods with left-Radau nodes.
- Do not retry the statement `IsCollocation ∧ HasRadauIANodes → IsAlgStable`
  unless the project changes the meaning of `IsCollocation` or the theorem
  assumptions are strengthened enough to exclude the counterexample.
